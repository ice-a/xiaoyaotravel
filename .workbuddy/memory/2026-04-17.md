## 修复前端 SyntaxError: Unexpected token '{'

**问题根因**：`renderGuide()` 函数中，`$("root").innerHTML = `...` `` 模板字符串的 HTML 部分结束后（`</section>`）缺少闭合反引号，导致后续的 JS 逻辑代码（1149-1331 行）全部被解析为模板字符串的纯文本内容。其中 1156 行的 `` api(`/api/...`) `` 中的反引号被误认为是外层模板字符串的结束符，后续 `{` 就触发了 `Unexpected token '{'` 语法错误。

**修复方案**：在评分 section 的 `</section>` 后添加 `` `; `` 关闭模板字符串，让后续 JS 逻辑代码正确脱离模板上下文。

**验证**：`node --check` 语法检查通过。

## 修复登录状态频繁掉线（2026-04-17 下午）

**根因分析**（3个问题）：

1. **`_keyHash` 未持久化到 `S.user`**（最关键）：会话恢复时 `/api/auth/me` 返回用户数据后，没有设置 `S.user._keyHash = saved.keyHash`，导致后续 `authHeaders()` 发送空 `X-User-Key`，所有认证 API 都返回 401
2. **网络异常直接清除会话**：`catch` 块无条件 `removeItem("xyy_session")`，网络波动/超时也会丢失登录态
3. **服务端重启清空内存缓存**：`userCache` 是纯内存 Map，重启后丢失，虽然代码有 DB fallback，但增加了失败风险

**修复内容**：

### index.html
1. **会话恢复**：改为直接用 `fetch` 调用 `/api/auth/me`（而非 `api` 函数），正确设置 `S.user._keyHash`
2. **401 精准清除**：只在服务端返回 401 时才清除 localStorage，网络错误/500 保留会话下次重试
3. **`ensureAuth()` 函数**：需要认证的操作前自动尝试从 localStorage 静默重新认证（带去重 Promise 防并发）
4. **`api()` 函数增强**：收到 401 时自动调用 `ensureAuth()` 尝试恢复，成功后自动重试请求（最多 1 次）；重试失败才标记为登出

## AdSense 广告单元 Bug 修复与优化（2026-04-17 下午）

### Bug 修复

1. **隐私政策/服务条款 Modal 样式丢失**：`class="modal"` 没有对应的 CSS 定义 → 改为 `class="modal-box"`（复用登录框的样式体系：白底、圆角、阴影、内边距）

2. **广告代码嵌入 JS 模板字符串中不执行**：`<script>(adsbygoogle = window.adsbygoogle || []).push({});</script>` 在 `innerHTML` 赋值时 `<script>` 标签被浏览器忽略 → 新增 `refreshAds()` 函数，在 `renderGuide()` 和 `renderSummary()` 的 innerHTML 赋值后手动调用 `adsbygoogle.push()`

3. **`data-ad-slot="auto"` 是无效值**：不是真实广告位 ID，AdSense 会填充空白或报错 → 改为占位符 `XXXXXXX`，`refreshAds()` 中检测并跳过占位符（等拿到正式 slot ID 后替换即可生效）
